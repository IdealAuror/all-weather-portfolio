## 缘起

全天候策略进入中国已有近十年讨论，但大多数实践停留在"参数调优"层面——桥水原版是 30% SP500 + 40% 长期国债 + 30% 大宗商品，到中国变成什么比例？

这个项目从 2026 年 4 月开始，经历了上百次回测、数十次否决、6 个策略变体的淘汰，最终沉淀出三套可落地策略。在这个过程中我们发现：**所有有效的决策都有一个共同点——它是从中国市场的实际数据出发，而不是从理论出发的。**

这个发现本身不是一个新发现，它是马克思主义中国化最核心的原则：**实事求是**。

本文件的目标不是给项目加上意识形态标签，而是把已经隐性的世界观显式化，让每个决策都能追溯到一个可讨论的哲学理由。这样，当未来的策略改动遇到分歧时，争论的不是"我觉得有效" vs "我觉得无效"，而是"这个改动是否坚持了实事求是"、"它抓住了当前的主要矛盾还是次要矛盾"。

## 一、四层结构

现有的 CLAUDE.md 前置原则是 Sperandeo 的三层结构。本框架在其下增加第 0 层哲学基座，形成：

```
第 0 层：哲学基座（实事求是 + 矛盾论 + 实践论）
     ↓ 指导但不替代
第 1 层：保障资本（Sperandeo）
第 2 层：一致性的获利能力
第 3 层：追求卓越的报酬
```

**第 0 层不是取代 Sperandeo，而是回答"为什么 Sperandeo 三层是对的"。**

- 为什么保障资本是第一层？因为资本安全是主要矛盾的前提出，没有资本就谈不上收益——这是矛盾论的基本判断。
- 为什么要从最简参数开始（奥卡姆剃刀）？因为实践论要求我们从感性认识出发，逐步上升到理性认识，跳过中间环节的复杂参数无法被实证检验。
- 为什么要写 memory？因为实践需要记录，记录才是认识发展的材料。

这种对应不是附会，而是说明三层结构之所以成立，是因为它符合更根本的认知规律。

## 二、概念映射表

### 2.1 实事求是

**哲学原文**："'实事'就是客观存在着的一切事物，'是'就是客观事物的内部联系，即规律性，'求'就是我们去研究。"（《改造我们的学习》，1941）

**译文**：每个参数回答"为什么这个值在中国数据上成立"。

| 项目实践 | 哲学对应 | 意义 |
|----------|----------|------|
| 30Y 国债三阶段合成 | 实事求是——不同数据条件用不同方法 | 不为了理论完美牺牲实证精度 |
| HS300 AND 抄底（非 SMA 趋势过滤） | 实事求是——A 股高波动，PE+价格双重确认更有效 | 不照搬桥水原版参数 |
| nonferr 75d 硬过滤（zero out） | 实事求是——中国大宗周期性远强于美国 | hard filter 比 soft filter 有效 |
| 20d 窗口优于 60d/15d | 实事求是——A 股换手率和波动特征决定的 | 非理论推导，是实证结果 |
| 提除 bond_10y | 实事求是——中国债市深度不足，bond_10y CACR -1.43pp | 理论觉得该有，实证说不要 |

**反面案例**（实践证明不实事求是的尝试）：
- HS300 trend filter：理论上第四个过滤器应该加强风控，但 A 股 HS300 趋势性不够强，CAGR -0.31pp 得不偿失
- SMA 斜率方向过滤：理论上斯波朗迪的 SMA 斜率确认能过滤假突破，但双重滞后让趋势过滤丧失了"迅速认赔"的能力

### 2.2 矛盾论 — 主要矛盾与矛盾转化

**哲学原文**："在复杂的事物的发展过程中，有许多的矛盾存在，其中必有一种是主要的矛盾，由于它的存在和发展规定或影响着其他矛盾的存在和发展。"（《矛盾论》，1937）

**译文**：决策的优先级——抓住影响最大的变量，忽略边际递减的优化。

| 项目实践 | 哲学对应 | 意义 |
|----------|----------|------|
| 80/20 检验：CAGR+0.3pp 或 MDD-1pp 以下不值得 | **主要矛盾的量化** | 量化"不值得做"的标准 |
| 桶结构（收益上限） vs 内部加权（微调） | **主要方面 vs 次要方面** | CAGR +0.41% 优于桶间风险分配带来的 0 |
| 牛市 CAGR 优先 → 熊市 MDD 优先 | **矛盾转化** | Sperandeo 三层本身就是矛盾转化的回应 |
| 否决不删：memory 保留失败记录 | **矛盾的特殊性** | 今天的主要矛盾，可能是明天的次要矛盾 |

**矛盾的特殊性**：
- nonferr 75d 趋势过滤有效，但 sp500 75d 效果弱，hs300 趋势过滤被否决——三个资产在同一窗口参数下结果不同，这就是矛盾的特殊性：不同质的矛盾用不同方法解决。
- 分层 RP 中 bucket_method 是主导参数（决定 CAGR/MDD 的 tradeoff），而 min_w 几乎无影响——同一个工具在不同参数位置上有不同权重。

### 2.3 实践论

**哲学原文**："实践、认识、再实践、再认识，这种形式，循环往复以至无穷，而实践和认识之每一循环的内容，都比较地进到了高一级的程度。"（《实践论》，1937）

**译文**：策略优化闭环 = 认识发展循环。

| 项目实践 | 哲学对应 |
|----------|----------|
| 问题定义 → 假设 | 感性认识→理性认识的飞跃 |
| 设计 → 验证 | 理性认识指导实践 |
| 稳健性检验 + 80/20 | 实践检验真理 |
| 决策 → 实施 → memory | 再认识，进入下一轮循环 |
| Read Before Write（行为原则第 8 条） | 没有调查就没有发言权 |

**关键含义**：memory 不是成功记录，而是实践记录。否决和失败和成功同样重要——它们是"再认识"阶段的输入材料。

7 步闭环和《实践论》的认识发展循环同构，不是巧合，是遵循同一认知规律的结果。

## 三、中国化演变史

全天候策略进入中国的改造过程，实事求是地从桥水原版一步一步演变为当前版本：

### 第一轮改造：资产可投性（2026-04）

| 桥水原版 | 中国版 | 原因 |
|----------|--------|------|
| SP500 直接持有 | SP500 通过 513500 ETF | QDII 通道，非直接市场 |
| 30Y 国债直接持有 | 30Y 国债三阶段合成 | 511130 2024 年才上市 |
| 商品直接做多 | gold/nonferr 通过 ETF | 商品期货 ETF 市场不完善 |
| 全球分散 | A 股 + 港股通范围 | 中国投资者的实际可投范围 |

### 第二轮改造：参数本地化（2026-05）

| 桥水原版参数 | 中国参数 | 实证依据 |
|-------------|----------|----------|
| 逆波动率常用 60d | 20d（保守增强/RP）或 60d（V3c） | 20d 更敏捷，A 股高波动需要更快响应 |
| max_w 无明确上限 | 0.25（逆波动率）/ 0.20（HRP） | 超出后集中度风险 > 收益 |
| 长期国债占比 ~40% | 无 bond_10y（V3-B RP/V3c） | CAGR -1.43pp，Sharpe 仅 -0.02 |
| trend filter 常用 SMA200 | SMA 75d | A 股趋势持续性差，更短的窗口更适合 |
| SP500 trend filter 效果显著 | nonferr 75d 才是唯一显著 | 中国大宗商品周期性强于美股 |

### 第三轮改造：风控机制原创（2026-05）

| 机制 | 来源 | 创新点 |
|------|------|--------|
| HS300 AND 抄底 | 原创，非桥水原版 | PB<30%ile + PE<30%ile + 价格回撤>25% + SMA120 上方 |
| D_excess 几何超额诊断 | 原创 | 不是桥水或主流指标，基于几何超额收益的尾部风险诊断 |
| 分层 RP 桶结构拆分 | 桥水 RP 框架上改造 | 5 桶（10Y/30Y 分拆）优于 4 桶，中国债市特殊性 |

### 第四轮改造：缩减与简化（2026-05 末至今）

| 尝试 | 结果 | 哲学解释 |
|------|------|----------|
| V3e/V3f 集中度变体 | Sharpe 2.60→2.37，否决 | 没有抓住主要矛盾（分散是全天候的核心） |
| 方案 C 动态补仓 | 不敌纯 100% RP，否决 | 储备金机会成本 > 择时收益，主要矛盾判断错误 |
| 桶间风险平价 | CAGR -0.13pp，否决 | 主要方面是桶结构，内部加权是次要方面 |
| Gold dip-buying（V3c/Con） | 关闭，零负收益年份 | 2013 年接飞刀，矛盾的特殊性判断错误 |
| 关 nonferr 趋势过滤 | 否决 | 最有效的单一风控，不可删 |

### 演变规律总结

1. **从照搬到改造**：桥水参数是起点，平均经过 2-3 轮实证检验后背离原版参数
2. **从加到减**：第一轮总是加东西（更多过滤器、更多参数），第四轮是删东西（关 dip、去 bond_10y）
3. **中国特殊性集中在两个领域**：商品周期（nonferr）和股票高波动（HS300）
4. **债务资产的特殊性**：中国债市深度、久期结构与美国不同，需要独立处理

## 四、动态更新机制

> 马克思主义不是教条，而是行动的指南。——恩格斯

本设计文档是活文档，不静止。具体维护规则：

### 4.1 memory 的哲学标签

在新增 memory 时，可选择标注矛盾类型标签：

| 标签 | 含义 | 示例 |
|------|------|------|
| `madx` | 主要矛盾 | 80/20 检验通过/不通过的决策 |
| `minx` | 次要矛盾 | 微调参数，影响不显著 |
| `spec` | 特殊性分析 | 某个资产与其他资产表现不同 |
| `psa` | 实事求是 | 数据推翻了理论预期 |
| `dial` | 矛盾转化 | 收益/风险优先级的切换 |

不强制修改已有 memory。新 memory 标注即可。

### 4.2 复审节奏

- **条件触发**：每累计 10 条新 memory，复审一次本文件
- **定期**：每季度（3/6/9/12 月）做一次结构复审
- **事件驱动**：当某个实证推翻了本文件的某个映射或预测时，立即更新

### 4.3 实证推翻理论的处理

如果出现以下情况：
- 某个标记为"不可删"的机制被回测否决
- 被标记为"major"的参数被更优值替代
- 中国化演变史中的某个"原因"被新的数据证伪

**处理流程**：
1. 更新本文件，在原文上做修订标记，保留旧记录（不删，加 `~~strike through~~` 或修订备注）
2. 在"演变史"末尾追加新的变化
3. 写一条 memory 记录推翻了什么、为什么、新的认识是什么
4. 这是最珍贵的更新——哲学框架被实践发展，不是被否定

### 4.4 与现有系统的关系

- `CLAUDE.md` 第 0 层 — 精炼摘要，日常决策引用
- `docs/framework/马克思主义.md` — 完整论述，复审时更新
- memory 标签 — 微操作级记录
- 七步闭环 — 实践论的操作化

三者层层递进，不重复。
