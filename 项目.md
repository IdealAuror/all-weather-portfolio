## 二、核心逻辑详解

### 2.1 数据加载（`data.py`）

**7 资产面板加载:**

| 资产 | 函数 | 数据来源 | 特殊处理 |
|------|------|----------|---------|
| hs300 | `_load_with_index()` | ETF(510300) + hs300_idx | 指数拼接 |
| us_sp500 | `_load_sp500_cny()` | ETF(513500) + sp500_idx×USDCNY | 汇率换算 |
| credit | `_load_with_index()` | ETF(511220) + credit_idx | 指数拼接 |
| bond_10y | `_load_bond_10y()` | ETF(511260) + treasury_idx + credit_idx | 三级回退 |
| bond_30y | `synthesize_bond_30y()` | 三阶段合成 | 久期放大/利差法/ETF |
| gold | `_load_gold_cny()` | ETF(518880) + 伦敦金×USDCNY + 持仓推算 | 三级回退+汇率 |
| nonferr | 手动拼接 | ETF(159980) + nonferr_idx + shfe_copper | 三级回退 |
| wti | `_load_wti_cny()` | SC原油 SC0 (2018+) + WTI USD×USDCNY proxy | 拼接，无QDII限制 |

**30Y 国债合成三阶段:**
1. 2015-01 ~ 2020-02：×3.0 久期放大（扣减 0.3%/年）
2. 2020-02 ~ 2024-03：利差法（10Y-30Y spread × duration 18.0）
3. 2024-03 ~ now：ETF 511130 真实 NAV

**安全扣减:** nonferr 0.5%/年, bond_30y 0.3%/年, wti 1.0%/年

### 2.2 回测引擎（`backtest.py`）

**主循环热路径（~5300 日 × 3 策略）:**
```
每日: h·rets → 组合收益, h 权重漂移修正, 更新 gold/hs300 峰值
每月(调仓日):
  1. 计算 trailing 窗口权重（逆波动率 或 HRP）
  2. 计算各 SMA（nonferr 75d, gold 75d, sp500 75d）
  3. HS300 AND 抄底检查（PB 分位 + PE 分位 + 价格回撤 + SMA120）
  4. _apply_trend_dip(): 趋势过滤 + 抄底 boost + max_w clip
```

**趋势过滤优先级:**
1. nonferr < SMA75 → 清仓→credit
2. us_sp500 < SMA75 → 清仓→credit
3. gold < SMA75 → 清仓→credit
4. HS300 AND 抄底（PB<30%ile + 回撤>25% + 价格>SMA120）→ boost 1.8x

### 2.3 风控模块（`risk.py`）

| 函数 | 作用 | 算法 |
|------|------|------|
| `inverse_vol_weights()` | 逆波动率加权 | 1/σ，clip→归一化迭代 |
| `hierarchical_rp_weights()` | 分层风险平价 | 桶内逆波动率 → 桶间等权/逆波动率 |
| `_precompute_percentile()` | PB/PE 分位预计算 | O(n²) 向量化 broadcast |
| `hs300_dip_check()` | HS300 AND 抄底 | PB 入场 + PE 出场 + 价格回撤 + SMA |
| `dynamic_cash_ratio()` | 动态现金比例 | 回撤 >20% → 0%，<5% → 30% |

### 2.4 统计模块（`stats.py`）

| 函数 | 复杂度 | 说明 |
|------|--------|------|
| `perf_metrics()` | O(n) | CAGR/vol/MDD/Sharpe/Calmar/D_excess |
| `block_bootstrap()` | O(s×b) | s=1000, b=1260d, 21d blocks |
| `d_significance()` | O(s×n) | s=2000 正态参数 Bootstrap |
| `risk_contribution_time_varying()` | O(m×w²) | m 调仓日 × w 天协方差矩阵 |
| `weight_stability()` | O(m×a) | 换手率/有效N/成本拖累 |

## 三、已知设计决策

### 已否决方案

- **方案C 70/30 动态补仓**: 不敌纯 RP 100%，补仓=摊平亏损
- **V3-B 桶间风险平价**: 桶等权最优，桶间风险分配无改善
- **HS300 trend filter**: 第四个趋势过滤器，CAGR -0.31pp 得不偿失
- **SMA 方向过滤**: 斯波朗迪 SMA 斜率确认反而全面恶化
- **估值型卖出信号**: PE+PB/市赚率/偏离度/PE95减半，六个尝试全部失败

### 关键设计约束

- **Bootstrap 代理权重**: 动态权重用最后调仓窗口固定权重近似
- **WTI 原油**: 已集成，但 QDII 限购暂不可执行
- **Sharpe 修正**: (CAGR - 2.2%) / vol，原始版保留为 sharpe_raw
- **div_idx / soymeal**: 已于 2026-05-27 移除

### 数据备份策略

数据有 3 层备份：本地 CSV → `data/` → GitHub 仓库，任何一层丢失可从另两层恢复。
