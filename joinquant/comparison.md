### 简化 #1 详解：HS300 PB/PE → 价格版

聚宽 `get_fundamentals()` 仅返回**季度**财报数据，无法做日频 PB/PE 分位计算。价格版替代逻辑：

```
入场: HS300 从高点回撤 >25% AND 价格 > SMA120 → 1.8× 加仓
出场: 恢复到 峰值−15% 以内 → 退出
```

保留原版核心思想——「极端便宜 + 趋势企稳时加倍」，用价格代理估值。V3c 受影响最小（60d 长窗口缓冲），保守增强最大（20d 短窗口 + nonferr 单趋势）。

## 四、差异模式分析

```
策略复杂度 ↑ → JQ-本地 差异 ↑

保守增强 (20d)        风险平价 (HRP)        V3c (60d)
   −1.26pp              −1.01pp             −0.66pp
   短窗口最敏感          链路最长              长窗口最稳健
```

- **窗口越短，月频影响越大** — 20d 逆波动率对窗口内波动率变化敏感，月频检查比日频多滞后
- **趋势过滤器越多，缓冲越足** — V3c 三重过滤互为冗余，单点时机偏差被分散
- **链路越长，累积误差越多** — RP 的四重趋势 + target_vol 多层叠加

## 五、结论

1. **聚宽版是本地版的适当简化**，核心算法（逆波动率/HRP + 趋势过滤 + 抄底）完全一致
2. **CAGR 偏差 0.66~1.26pp**，方向一致，三策略排序不变
3. **所有策略在聚宽版均达标**（各自的 CAGR/MDD 约束）
4. **HS300 PB/PE → 价格版是唯一实质性简化**，受聚宽数据 API 限制，无法避免
5. **聚宽版适用于 2020 年以后的回测验证**，更早期需用本地引擎
